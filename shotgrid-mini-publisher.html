Flow/PT (ShotGrid) Mini Publisher

Standalone PySide6 publisher that creates ShotGrid Versions, optionally uploads media/files, and writes a structured local publish folder with a manifest.

Python PySide6 Flow PT / ShotGrid Pipeline Tools

Problem

Publishing work to Flow Production Tracking (ShotGrid) often requires repetitive manual steps like choosing the correct context (Project/Shot/Task), validating inputs, creating a Version, uploading media, and keeping local publish output consistent. Without validation and structure, bad publishes can slip into tracking systems and create downstream confusion.

Solution

I built a lightweight standalone "mini publisher" with a guided workflow. It automates Version creation and optional uploads, while also writing a structured local publish folder and manifest.json for traceability.

Key Features

Context Selection and Workflow

  • Connects to Flow PT / ShotGrid and loads available Projects.
  • Browses Project -> Shot -> Task to establish publish context.
  • Guided publish flow designed for repeatable, productionstyle usage.

Validation and Safety

  • Prepublish validation step to catch missing or invalid inputs.
  • Validation errors block publishing to prevent bad tracking data.
  • Warnings are shown clearly while allowing a publish when appropriate.

Version Creation and Uploads

  • Creates a ShotGrid Version linked to the selected Shot and Task.
  • Optional thumbnail upload and optional movie upload.
  • Optional file attachments uploads (schema-aware).

Local Publish Output

  • Creates a structured publish folder under a configurable publish root.
  • Writes manifest.json describing publish metadata and files.
  • Designed to mirror "publish drop" conventions used in production pipelines.

Optional PublishedFile Support

  • Attempts PublishedFile creation when enabled (optional/nonblocking).
  • Handles local storage/path constraints gracefully without failing the main publish.

Demo Mode (DRY_RUN)

  • DRY_RUN simulates API actions without requiring credentials.
  • Still generates local publish output to demonstrate end-to-end workflow.

How to Run

Install & Launch

python -m venv .venv
# Windows:
.venv\Scripts\activate
pip install -r requirements.txt
python main.py

Environment Config

Configure connection + output settings via .env (e.g., URL, script name, API key, publish root).

Demo Mode

# In .env (example)
DRY_RUN=1
PUBLISH_ROOT=C:\TEMP\flow_publishes

python main.py

Outputs

  • ShotGrid entities: creates a Version. Optionally uploads thumbnail/movie/files and creates PublishedFile.
  • Local publish folder: structured output under PUBLISH_ROOT.
  • manifest.json: records publish metadata, file list, and paths for traceability.

Design Notes

  • Design intent: A minimal, standalone publisher that demonstrates core ShotGrid publish concepts (context -> validate -> publish -> register -> local drop) without studio specific dependencies.
  • Tradeoff: PublishedFile creation is optional/non-blocking because ShotGrid storage configuration varies by studio and can fail on path constraints.
  • Practicality: DRY_RUN enables demo/testing in environments where API credentials or storage mappings aren’t available.
