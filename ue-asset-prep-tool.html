UE Asset Prep Tool

Unreal Engine editor utility for preparing assets for production, validating static meshes, enforcing import standards, and exporting structured JSON reports to support consistent pipelines and faster iteration.

Unreal Engine Editor Utility Validation JSON Reports

Problem

In Unreal projects, inconsistent imports and asset quality issues (missing collision, bad LOD setups, invalid naming, oversized textures, incorrect folder placement) create gameplay bugs, performance problems, and downstream rework. Teams need a repeatable "preflight" pass inside the editor to validate assets before they hit production content branches.

Solution

I built an Unreal Editor utility that scans selected assets (or an entire folder), runs a checklist of production style validations (LOD/collision/naming/location rules), presents results in a clear UI, and writes a structured JSON report. The tool is designed to be lightweight, editor native, and suitable for iterative "scan -> fix -> rescan" workflows.

Key Features

Asset Selection and Scope

  • Run on selected assets or scan an entire content folder.
  • Targets Static Mesh validation as a core production use-case.
  • Designed to extend to other asset types (textures/materials) as needed.

Static Mesh Validation

  • LOD checks: validates LOD presence and basic configuration expectations.
  • Collision checks: detects missing collision and flags non compliant setups.
  • Naming rules: validates naming conventions for asset types/prefixes.
  • Placement rules: checks folder placement under expected content paths.

Reporting

  • Severity based results (ERROR/WARNING/INFO).
  • Exports a structured JSON report for tracking, review, or CI ingestion.
  • Report supports pipeline "proof" for content readiness.

Workflow and UX

  • Editor native UI built for fast iteration: scan -> review -> fix -> rescan.
  • Designed to reduce context switching (fix issues inside the editor).
  • Clear output messaging for artists and technical users.

Configurable Rules (Pipeline-Friendly)

  • Rules can be tuned perproject (naming, folder paths, validation thresholds).
  • Supports consistency across teams by keeping standards explicit.

How to Run

Install

  1. Clone/download the repo.
  2. Copy the tool into your project (commonly into Content/Editor or a Plugins folder depending on implementation).
  3. Open the project in Unreal Engine.

Launch

  1. Open the Editor Utility Widget (or tool entry point provided by the project).
  2. Select a folder or assets to scan.
  3. Run validation and review results.
  4. Export JSON report if needed.

Note: If your repo provides a specific Editor Utility Widget name/path or plugin module name, replace the generic install/launch steps above with the exact instructions.

Outputs

  • In editor results: list of assets with issues grouped by severity.
  • JSON report: exported file containing per asset validation results and summary counts.
  • Optional: logs suitable for debugging validation rules and thresholds.

Design Notes

  • Design intent: An editor native "asset preflight" tool that helps teams enforce content standards early, improving performance stability and reducing downstream rework.
  • Tradeoff: Rules must remain configurable hard coded validation becomes noisy or incorrect across different projects, so the tool is designed to be tuned per studio/project.
  • Pipeline fit: JSON reporting supports asset QA signoff, content review, and future automation/CI checks.
