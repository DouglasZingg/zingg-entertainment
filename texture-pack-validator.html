Texture Pack Validator

Standalone Python and PySide6 utility for validating game/VFX texture exports (Substance Painter / game ready sets), auto fixing common naming issues, and generating severity based QA reports.

Python PySide6 Pillow Texture QA

Problem

Texture deliveries often fail silently because if missing required maps, inconsistent naming, wrong file formats, and nonproduction safe resolutions lead to broken materials, wasted review cycles, and late-stage rework. Teams need a fast, repeatable way to validate texture exports before handoff to Unreal/Unity/VFX pipelines.

Solution

I built a standalone validator that scans an export folder (or multiple folders in batch), applies profile aware rules (Unreal / Unity / VFX), reports issues by severity (ERROR/WARNING/INFO), and can optionally auto fix naming to match expected conventions. Results can be exported to JSON/HTML for review and tracking.

Key Features

Profile-Aware Validation

  • Supports validation profiles for Unreal, Unity, and VFX delivery rules.
  • Required map rules vary by profile (e.g., BaseColor and Normal always required. ORM/individual channels depending on target).
  • Severity based results: ERROR (blocks delivery), WARNING (needs review), INFO (notes).

Required Maps and Naming QA

  • Detects missing required maps for an asset set (BaseColor/Normal plus ORM or AO/Roughness/Metallic based on profile).
  • Identifies naming issues that prevent correct map classification (type parsing and version suffix handling).
  • Optional auto fix naming to rename files into expected conventions (batch safe option available).

Image Metadata and Format Checks (Pillow)

  • Reads resolution and warns on nonpower of two textures (where applicable).
  • Warns/errors on oversized textures based on configured max size thresholds.
  • Validates extension expectations (VFX can allow EXR more broadly).
  • Performs basic channel sanity checks (e.g., expected channel counts).

ORM Packed Texture Validation

  • Validates packed ORM textures: must be RGB (3 channels).
  • Warns on unexpected alpha channels.
  • Warns when channels look suspiciously flat/identical (common export mistakes).

Batch Scan and Watch Mode

  • Batch scanning across multiple folders with an aggregated summary export.
  • Optional "watch folders" mode to auto rescan on changes (best effort, debounced).

Reporting

  • Exports folder reports to report.json and report.html.
  • Exports batch summaries to batch_report.json for pipeline tracking.

How to Run

Quick Start (Windows CMD)

setup.bat
python main.py

Quick Start (macOS/Linux)

chmod +x setup.sh
./setup.sh
python main.py

Usage

  1. Select an export folder (or add multiple folders for batch).
  2. Choose Profile (Unreal / Unity / VFX).
  3. Scan and review issues by severity.
  4. Export JSON/HTML reports for handoff and review.

Tests (Optional)

pip install -r requirements-dev.txt
pytest -q

Outputs

  • Per-folder: <export_folder>/reports/report.json and <export_folder>/reports/report.html
  • Batch: <selected_folder or home>/reports/batch_report.json
  • Optional: renamed files when auto-fix naming is enabled (to align with expected map conventions)

Design Notes

  • Design intent: A practical QA step for texture deliveries with fast feedback, consistent rules, and exportable reports that fit into real production handoffs.
  • Tradeoff: Auto fix naming is useful but intentionally optional. Renaming files changes source so it’s gated behind explicit user control (and batch renaming requires opt-in).
  • Pipeline fit: Profiles (Unreal/Unity/VFX) keep rules explicit and avoid “one-size-fits-all” validation that produces noisy or misleading results.
