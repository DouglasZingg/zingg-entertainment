Pipeline Delivery Packager

Standalone packaging utility that builds clean delivery drops from a source folder, validates required structure, generates a manifest, and optionally hashes files for integrity checks. Designed to mirror real "vendor delivery/handoff" workflows in game & VFX pipelines.

Python Packaging Manifest and Hash Pipeline Tools

Problem

Deliveries often fail for avoidable reasons. Without a manifest and integrity checks, handoffs are hard to audit and issues are discovered late after transfers, uploads, or vendor integration.

Solution

I built a standalone "delivery packager" that validates a source folder against expected delivery rules, plans a copy, then builds an output package with a structured folder layout and a detailed manifest.json. Optional hashing provides integrity verification and supports "trust but verify" handoffs.

Key Features

Planning and Validation

  • Validates required folders/files before packaging (errors block output).
  • Separates validation results into ERROR vs WARNING for clear triage.
  • Preview/plan step to show what will be copied and where (safe “no surprises” workflow).

Packaging and Structure

  • Builds a clean output delivery folder with consistent structure.
  • Category routing (organizes files into expected destinations based on rules).
  • Collision handling to avoid accidental overwrites.
  • Supports "skip existing" vs "overwrite" behavior (user-controlled).

Manifest an Integrity

  • Generates manifest.json describing all packaged files (src->dst mapping, sizes, timestamps).
  • Optional file hashing (e.g., SHA-256) written into the manifest for integrity checks.
  • Detects mismatches between expected and actual content (helps catch bad handoffs early).

Batch Friendly

  • Designed to run consistently for repeated deliveries (predictable output).
  • Clear logs and structured outputs suitable for CI or pipeline integration.

How to Run

Quick Start

python -m venv .venv
# Windows:
.venv\Scripts\activate
pip install -r requirements.txt
python main.py

Usage

  1. Select or enter a source folder (the work you want to deliver).
  2. Select an output destination (delivery root).
  3. Run validation and review errors/warnings.
  4. Preview the plan (optional but recommended).
  5. Package and verify output and manifest.

Tests (Optional)

pip install -r requirements-dev.txt
pytest -q

Outputs

  • Delivery folder: a clean packaged output with organized structure.
  • manifest.json: file inventory with src→dst mapping, metadata, and optional hashes.
  • Logs: validation and packaging results suitable for review or automation.

Design Notes

  • Design intent: Provide a repeatable, auditable delivery step that catches packaging problems early and produces predictable, studio friendly handoff outputs.
  • Tradeoff: Strict validation reduces surprise failures downstream, but requires clear rules and sometimes blocks delivery until issues are resolved.
  • Practicality: The manifest and hashing supports "what changed?" and "did transfer corrupt anything?" questions during review.
