USD Inspector & Packager

Standalone pipeline utility for inspecting USD assets and packaging dependencies into a clean, portable drop. Designed to demonstrate OpenUSD (pxr), dependency scanning, validation reporting, and batch packaging.

Python OpenUSD (pxr) Pipeline Tools Batch and Packaging

Problem

USD assets frequently depend on external files (sublayers, references, payloads, textures). Missing or broken dependencies cause downstream failures in DCCs, render farms, and engine ingestion.

Solution

I built a standalone inspector that scans USD files for dependency issues, produces severity-ranked results, and a packager that collects dependencies into a clean folder structure with a detailed manifest.

Key Features

Dependency Inspection

  • Scans USD assets for sublayers, references, payloads, and texture dependencies.
  • Supports .usd, .usda, .usdc, and .usdz.
  • Flags missing/unresolved/invalid dependencies with severity (ERROR, WARNING, INFO).
  • Exports structured results to report.json.

Packaging & Portability

  • Packages USD files and resolved dependencies into a clean, organized output folder.
  • Generates manifest.json describing copied files, source paths, destination paths, and metadata.
  • Collision safe copying to prevent overwrites when names conflict.
  • Optional SHA-256 hashing for integrity verification.

Portable Mode

  • Rewrites asset paths inside packaged USD files to use packaged-relative paths.
  • Produces self contained, portable asset drops suitable for review, vendor handoff, or archival.

Batch Processing

  • Scans and packages entire folders of USD files.
  • Writes batch_summary.json with per file results and aggregate counts.
  • Automation friendly workflow for large asset sets.

How to Run (Windows)

Recommended Python: 3.10 or 3.11 (OpenUSD builds vary by Python version).

setup.bat
run_app.bat

Note: pxr is not pure Python; the project keeps requirements minimal because OpenUSD installs vary by platform. If from pxr import Usd fails, confirm you’re using the venv Python and that OpenUSD DLLs are discoverable on PATH.

Demo Samples

The repo includes a demo generator to create sample USD files and textures for testing scans and “missing dependency” scenarios. 

python tools\make_demos.py

Outputs

  • manifest.json - copied files, src->dst mapping, types/sizes, optional sha256, missing/skipped deps
  • report.json - scan results, counts, metadata
  • batch_summary.json - per-file status and totals (batch packaging)
