Maya Asset Validator

Production style validation tool for Autodesk Maya that scans scenes for common pipeline issues, provides severity ranked results, and offers guided auto fix actions to prepare assets for export and downstream ingestion.

Python Maya PySide6 / PySide2 Validation and Auto-Fix

Problem

In game/VFX pipelines, Maya scenes frequently accumulate issues that cause broken exports and downstream failures: naming drift, non zero transforms, bad history, missing textures, invalid geometry, and inconsistent conventions. These problems are often discovered late (during export, lookdev, rigging, or engine ingestion), creating rework and churn.

Solution

I built a Maya native validator that performs a comprehensive scan across naming, transforms, geometry, and textures. Findings are grouped by severity (ERROR/WARNING/INFO) and presented in a clear UI with targeted "auto-fix" actions. The tool is designed to be fast, repeatable, and usable by both artists and TDs.

Key Features

Validation Coverage

  • Naming checks: duplicate names, invalid characters, naming conventions.
  • Transform checks: non zero transforms, frozen transforms, pivot expectations (configurable).
  • Geometry checks: non manifold geometry, lamina faces, bad normals, history issues.
  • Texture checks: missing file paths, broken connections, common texture pipeline pitfalls.

Severity Based Reporting

  • Findings categorized as ERROR, WARNING, and INFO.
  • Errors block a "ready for export" state. Warnings highlight risk. Info provides helpful notes.
  • Designed for fast triage and consistent standards enforcement.

Auto Fix Actions

  • Optional auto fix operations for common issues (safe, targeted actions).
  • Fix flows designed to be explicit and auditable (users choose what to fix).
  • Supports iterative "scan -> fix -> rescan" workflow.

UI and Compatibility

  • Qt UI designed for Maya: PySide6 (newer Maya) with fallback compatibility for PySide2/PySide.
  • Targets Maya 2020–2026 style environments.
  • Organized modular checks (easy to extend with new rules).

Artist-Friendly Workflow

  • Clear issue lists with actionable descriptions.
  • Designed to be usable without deep technical knowledge.
  • Emphasizes predictable standards and export readiness.

How to Run

Run Inside Maya

  1. Clone/download the repo.
  2. Point Maya’s Python path at the project folder (or add it via a shelf tool).
  3. Run the launcher / entry script.

Typical Launch Snippet

# In Maya Script Editor (Python)
import sys
sys.path.append(r"C:\path\to\maya-asset-validator")
import main
main.run()

If your repo uses a different entry point (e.g., maya_launcher.py), update the import accordingly.

Outputs

  • In app results: grouped issue lists with severity and descriptions.
  • Auto fix results: scene modifications applied by selected fixes (non-destructive patterns where possible).
  • Optional reports: if enabled, export JSON/HTML style summaries for review and tracking.

Design Notes

  • Design intent: A "preflight" validation step for Maya assets that makes pipeline standards visible and repeatable, with clear severity and optional fixes.
  • Tradeoff: Auto fix operations are intentionally conservative some issues are better flagged for artist review than silently changed.
  • Extensibility: Checks are modular so studios can add/disable rules or tune severities without rewriting the UI.
