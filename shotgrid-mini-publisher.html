Flow/PT (ShotGrid) Mini Publisher

A standalone PySide6 desktop tool that connects to Autodesk Flow Production Tracking (ShotGrid) to create a Version publish, upload media/files, and generate a local publish folder with a manifest.

Python PySide6 ShotGrid / Flow PT Pipeline Tools

Problem

Publishing to ShotGrid often involves repetitive steps: selecting context (Project/Shot/Task), validating inputs, creating Versions, uploading media, and keeping a clean local publish structure.

Solution

I built a small "mini publisher that guides users through the publish workflow, blocks publishes on validation errors, and automates Version creation and optional uploads while also writing a local publish folder and manifest.json.

Key Features

  • Connect and browse Project -> Shot -> Task
  • Build a publish request (name/version/comment/files)
  • Validation step (errors block publish)
  • Create a ShotGrid Version linked to Shot and Task
  • Optional thumbnail/movie and file uploads (schema-aware)
  • Create local publish folder structure and write manifest.json
  • Optional PublishedFile creation (non-blocking)
  • DRY_RUN demo mode to test UI/workflow without credentials

How to Run

Real ShotGrid / Flow PT

python -m venv .venv
# Windows:
.venv\Scripts\activate
pip install -r requirements.txt
python main.py

Configure environment variables in a .env file (URL, script name, API key, publish root).

Demo Mode (No credentials)

The tool supports a demo workflow that simulates Version creation/uploads while still generating the local publish output. 

# In .env:
DRY_RUN=1
PUBLISH_ROOT=C:\TEMP\flow_publishes

python main.py

Testing / Walkthrough

  1. Launch the app -> Connect & Load Projects
  2. Select Project, Shot, Task
  3. Enter Publish Name and Version
  4. Add file(s) -> Validate -> Publish
  5. Confirm PUBLISH_ROOT/.../v###/manifest.json exists

Production Notes

  • Schema-aware uploads and optional PublishedFile creation
  • Designed to be usable even without ShotGrid access via DRY_RUN
  • Common issue: local storage/path field constraints in ShotGrid (documented in troubleshooting)
