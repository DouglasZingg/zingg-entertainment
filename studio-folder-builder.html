Studio Folder Builder (Template Generator)

Standalone Python and PySide6 tool that generates a studio style project folder structures from configurable templates, with preview first planning, safe build, and a reproducible manifest. Optional Flow/PT (ShotGrid) integration.

Python PySide6 Templates Flow/PT Optional

Problem

New shows and projects often start with inconsistent folder layouts.That inconsistency wastes time, breaks automation, and makes onboarding harder. Teams need a repeatable way to generate a clean “studio standard” folder tree for shots and assets without risking accidental filesystem damage.

Solution

I built a template driven folder generator with a "Preview Plan -> Build" workflow. Users choose a root, template, and project name, then define either sequences/shots or asset categories/assets. The tool previews exactly what will be created, performs a safe build step, and writes a manifest for auditing and reproducibility. Flow/PT integration is optional for studios that want to drive sequences/shots from production tracking.

Key Features

Template Based Structure

  • Templates define project level folders and per shot/asset trees.
  • Supports starter files (entries ending in .md / .json are created as files).
  • Templates are editable and live in the repo templates/ folder.

Two Production Modes

  • Shots Mode: generates sequences/SQ###/SH###/... style trees.
  • Assets Mode: generates assets/<category>/<asset>/... style trees.

Preview Plan and Safe Build

  • Preview Plan shows exactly what will be created before touching the filesystem.
  • Build step creates folders/files and tracks created, skipped or, errors counts.
  • Designed to reduce risk (preview first, explicit build action).

Reproducibility

  • Save/Load job configs to rerun identical builds later (reproducible project setup).
  • Manifest output provides an audit trail of what was created and what was skipped.

Optional Flow/PT (ShotGrid) Integration

  • Optional dependency set for connecting to Flow/PT and loading sequences/shots.
  • Uses a local flow_config.json (gitignored) for credentials and project ID.
  • Supports studios with custom fields (troubleshooting guidance included in repo docs).

How to Run

Quick Start (Windows)

setup.bat
run_app.bat

Manual Setup (Windows)

python -m venv .venv
.venv\Scripts\activate

pip install -r requirements.txt
pip install -r requirements-dev.txt

python -m pytest -q
python main.py

Optional: Enable Flow/PT Integration

.venv\Scripts\activate
pip install -r requirements-flow.txt

Then copy flow_config.example.json to flow_config.json and fill in your URL/script credentials/project ID. (Do not commit flow_config.json — it contains secrets.)

Usage

  1. Select a Root directory (example: D:\shows).
  2. Select a Template and enter a Project name.
  3. Choose Shots Mode or Assets Mode.
  4. Click Preview Plan -> verify output -> click Build.
  5. Optionally Save Config to rerun the same job later.

Outputs

  • Folder tree: project/shot/asset structure created under your selected root.
  • manifest.json: written after Build (includes template name/version, timestamp, counts, and per item outcomes).
  • Job config (optional): saved via Save Config / Load Config for reproducible builds.

Design Notes

  • Design intent: Create studio consistent folder structures with a preview first workflow, minimizing risk while maximizing reproducibility.
  • Tradeoff: Templates are intentionally explicit (not “auto inferred”) so studios can enforce structure, even if that means more up front template authoring.
  • Practicality: Flow/PT integration is optional to keep the tool usable for anyone without credentials, while still demonstrating production tracking integration when available.
