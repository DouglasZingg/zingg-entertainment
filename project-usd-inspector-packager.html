USD Inspector & Packager

Standalone pipeline utility that scans USD assets for dependency issues and packages resolved dependencies into a clean, portable drop with reports and manifests.

Python OpenUSD (pxr) Pipeline / VFX Batch and Packaging

Problem

USD assets commonly reference external files (sublayers, references, payloads, textures). When dependencies are missing or paths are brittle, assets fail to load downstream in DCCs, render farms, and engine ingestion often discovered late, after handoff or transfer between machines.

Solution

I built a standalone inspector that scans USD files for dependency issues and produces severity ranked results, plus a packager that collects resolved dependencies into a consistent folder structure with a detailed manifest. An optional “portable mode” rewrites paths inside packaged USDs to make the result self contained.

Key Features

Dependency Inspection

  • Supports .usd, .usda, .usdc, and .usdz.
  • Scans for sublayers, references, payloads, and texture dependencies.
  • Flags missing/unresolved/invalid dependencies with severity (ERROR, WARNING, INFO).
  • Exports structured scan results to report.json.

Packaging & Manifests

  • Packages USD files and resolved dependencies into a clean output folder structure.
  • Writes manifest.json with src->dst mapping, file metadata, and missing/skipped dependencies.
  • Collision safe copying to prevent overwrites when file names conflict.
  • Optional SHA-256 hashing for integrity verification.

Portable Mode

  • Rewrites asset paths inside packaged USD files to use packaged relative paths.
  • Produces self contained drops suitable for vendor handoff, review, or archival.

Batch Processing

  • Scans/packages folders of USD assets in one run.
  • Generates batch_summary.json with per file status and aggregate totals.
  • Automation friendly workflow for large asset sets.

Demo & Validation Utilities

  • Includes demo/sample generators to validate common dependency scenarios.
  • Useful for testing “missing vs resolved” cases and demonstrating tool behavior.

How to Run

Quick Start (Windows)

setup.bat
run_app.bat

OpenUSD (pxr) is not pure Python and installation varies by platform/Python version. If from pxr import Usd fails, confirm the correct venv Python is being used and that OpenUSD binaries are discoverable (e.g., PATH on Windows).

Generate Demo Assets

python tools\make_demos.py

Outputs

  • report.json: scan results, dependency findings, and severity counts.
  • manifest.json: package contents including src→dst mapping, file metadata, optional sha256, missing/skipped deps.
  • batch_summary.json: per-file results and aggregate counts for batch operations.
  • Packaged folder: organized output containing USD(s) and collected dependencies.

Design Notes

  • Design intent: Offline validation and packaging step that prioritizes portability and downstream reliability over preserving original authoring directory layouts.
  • Tradeoff: Portable mode improves handoff success but can diverge from an artist’s original path structure.
  • Practicality: Manifests and optional hashing support auditing, review, and automation checks.
